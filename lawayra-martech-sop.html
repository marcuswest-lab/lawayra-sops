Section 04

Source Attribution — How It Works

How we figure out which marketing channel each booking came from, and why email matching is critical.

  1. Guest books on BookingLayer and is asked "How did you hear about us?"
  2. BookingLayer sends a daily CSV with their answers to admin@lawayra.com. An n8n workflow parses it and fills the Self-Reported Source column (col I).
  3. A separate BookingLayer report sends guest emails. Another n8n workflow parses it and fills the Email column (col F).
  4. A nightly Apps Script cross-references each guest's email against the GG Leads sheet (4,200+ Google Ads lead form emails).
  5. If the email matches a Google Ads lead → Source Category = "Google Ads" (overrides whatever they self-reported).
  6. If the email does not match and they said "Google Search" → Source Category = "Organic Search/SEO".
  7. All other answers are mapped to standard categories (Friend Rec → Word of Mouth, Reddit → Reddit, etc.).
Why email matching matters: People who clicked a Google Ad often say "Google Search" when asked. Name matching misses about 57% of Google Ads bookings because people use different names on ad forms vs BookingLayer. Email is the reliable identifier.
Section 05

What's Still Manual

A few things still require a human touch.

Sales tab amounts: Bookings with status "deposit_paid" show the deposit amount, not the full price. The full amount needs to be entered manually once the guest has fully paid.

New source types: If a guest writes something new that does not exist in the mapping (e.g., "AyaAdvisors"), it needs to be added to the category mapping in the Apps Script.

GG Leads sheet: This is auto-populated by Google Ads lead forms. No manual action needed — listed here for context only.

Section 09

How to Send Email Broadcasts

Step-by-step process for sending email broadcasts to the LaWayra subscriber list via GetDrip.

Email Template Reference

Template file: community-call-email.html in the project repo

Template includes: Subscriber first name personalization, 5 timezone display, RSVP link, call format overview

Drip account: admin@lawayra.com

Walkthrough video: Loom — How to send a broadcast in Drip

Broadcast Process

  1. Draft the email — Write copy with event details (date, time in multiple time zones, RSVP link to Circle event). Use the community call email template at community-call-email.html as reference for formatting.
  2. Review in WhatsApp group — Share the draft in the team WhatsApp group and get feedback before sending.
  3. Send a test email — Send to yourself first. Check formatting, links, time zones, and mobile rendering.
  4. Broadcast via Drip — Log in to getdrip.com. Select the appropriate list/segment. Compose or paste your email. Send the broadcast. See the Loom walkthrough for a visual guide.
  5. Confirm delivery — Check the Drip dashboard for open rates and click rates after sending.
Best practices:
• Always include times in at least 3 time zones (CDT/COT, EDT, BST).
• If multiple events are close together, combine into one email to avoid fatigue.
• Day-before reminder emails improve attendance significantly (lesson learned: 18 RSVPs but only 5 showed up without reminders).
• Never send without team review first.

How to Build & Launch a Funnel Page

When LaWayra launches a new program, workshop, or offer, it needs its own landing page. We use Claude Code to generate branded, self-contained HTML pages that are hosted on GitHub Pages — no WordPress or external builder needed.

Build Process

  1. Gather content — Collect: program name, headline/tagline, start date & schedule, price, checkout URL, program description, what's included, facilitator photo + bio, hero image, FAQ (3–5 questions). The more complete, the faster the build.
  2. Place source files in the project folder — Put PDFs (proposals, mockups), promo slide PNGs, and photos in the project directory. Claude Code can extract text from PDFs and read images directly.
  3. Run the skill — Open Claude Code and run /build-landing-page. Provide the details and file locations. Claude Code will read source files, extract content, apply LaWayra branding, and generate a self-contained HTML page.
  4. Review the page — Open the HTML file locally in your browser. Check: content accuracy, mobile responsiveness, facilitator photo, CTA button links, logo rendering. Request specific changes ("make the heading bigger", "swap the hero image", "change the price").
  5. Push to GitHub — Tell Claude Code to push. It will update .gitignore, commit, and push to origin main. The page goes live on GitHub Pages within 1–2 minutes.
  6. Share the viewable URL — Format: https://marcuswest-lab.github.io/lawayra-content-strategy/[filename].html. Share with team for final review and sign-off.
Brand specs for landing pages:
• Font: Poppins (Google Fonts) — Bold (700) for headings, Regular (400) for body
• Colors: Olive #3F4021, Cream #FFF7E7, Amber #E4A017 (CTAs), Charcoal #2B2728
• Logo: lawayra-logo-new.png (outline flower, transparent background)
• Voice: Warm, authentic, compassion-first — reference content-machine/lawayra-voice-guide.md
• Full SOP: lawayra-landing-page-sop.md in the project repo

Section 11

How to Set Up Claude Code

Claude Code is the AI-powered CLI tool that powers content generation, landing page building, dashboard creation, and automation for LaWayra. A new team member needs to set it up on their machine to use the slash commands (skills) documented in this SOP.

Setup Process

  1. Install Claude Code — Visit claude.ai/code or install via terminal: npm install -g @anthropic-ai/claude-code. Requires Node.js 18+.
  2. Authenticate — Run claude in terminal. It will prompt you to sign in with your Anthropic account. Follow the browser-based auth flow.
  3. Clone the project repo — git clone https://github.com/marcuswest-lab/lawayra-content-strategy.git and cd into the directory.
  4. Verify setup — Run claude inside the project directory. It should load the CLAUDE.md project instructions automatically. Type /help to see available commands.
  5. Test a skill — Try /create-facebook-post with a test hook to verify the skills are working.
